# SuperK Core (Superpowers + Karpathy)

SuperK fuses workflow (Superpowers) with behavioral discipline (Karpathy).

## The Karpathy Checkpoint (RUN THIS BEFORE EVERY ACTION)
1. **ASSUMPTIONS** → State them. If uncertain, ASK.
2. **SIMPLICITY** → Find the simplest way. No speculative features.
3. **SCOPE** → Touch only what's necessary. Every line must trace to the task.
4. **CRITERIA** → Define verifiable success. Loop until verified.
*If any fail → STOP and fix.*

## Workflow Priority
1. **KARPATHY CHECK** (Always first)
2. **brainstorming** (Design before code)
3. **writing-plans** (Plan tasks + estimates)
4. **subagent-driven-development** (Execute tasks with review)
5. **requesting-code-review** (Spec & Karpathy compliance)
6. **verification-before-completion** (Prove it works)
7. **retro-learner** (Capture lessons)

## Red Flags (Do not rationalize skipping skills)
- "This is just a simple question" → Questions are tasks. Use skills.
- "I need more context first" → Skills tell you HOW to get context.
- "I know what that means" → Knowing ≠ using. Invoke the skill.
- "While I'm here, let me improve..." → Scope creep (Karpathy Rule 3).

## Instruction Priority
1. User explicit instructions > 2. SuperK skills > 3. Default system prompt.
